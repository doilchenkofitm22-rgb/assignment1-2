 # Процес SAST (статичне сканування коду)

## Налаштування

1. **CI/Гітлаб або GitHub Actions** запускає робочий процес з файлу `.github/workflows/sast.yml`. При кожному коміті або PR виконується:
   - `pip install -r requirements.txt` для встановлення залежностей;
   - запуск `semgrep --config p/ci` для перевірки Python‑коду на відомі шаблони уразливостей;
   - запуск `bandit -r src/` для пошуку небезпечних викликів Python;
   - запуск `pip-audit` для оцінки залежностей.
2. Результати зберігаються як артефакти у збірці та/або як анотації до PR.



- Для кожної знайденої знахідки визначається рівень критичності на основі ймовірності експлуатації та впливу (CVSS).  
- Знахідки `High` або `Critical` автоматично перетворюються на завдання в системі відстеження (GitHub Issues) з міткою `security` та прив’язкою до відповідного коміту.
- Команда development аналізує та виправляє проблему у окремій гілці, посилаючись на issue.
- Після виправлення запускається повторний SAST для перевірки закриття уразливості.

## Відстеження та звітність

- Звіти Semgrep/Bandit завантажуються в артефакти CI для подальшого аудиту.
- Пріоритезація виправлень фокусується на уразливостях, що дозволяють віддалене виконання коду, SQL‑інʼєкції, XSS та витоки даних.
