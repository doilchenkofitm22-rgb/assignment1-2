
# SAST: процес керування

1. CI запускає `semgrep`, `bandit`, `pip-audit` на кожен PR.
2. Результати публікуються як артефакти/анотації в PR.
3. Для `High/Critical` автоматично створюємо Issue з тегом `security`.
4. Пріоритезація: CVSS, експлуатованість, бізнес-цінність.
5. Виправлення: посилання на коміт/PR у відповідному Issue.
